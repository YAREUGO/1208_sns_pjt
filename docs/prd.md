 Mini Instagram

문제 정의: 개인의 일상이나 감정을 기록하고 공유할 수 있는 SNS는 많지만, 대부분은 알고리즘 중심의 과도한 노출과 피로감이 존재한다. 개인이 단순히 ‘지금의 기분’을 기록하고, 친숙한 인터페이스 안에서 가볍게 소통할 수 있는 공간이 부족하다.

왜 중요한가: 단기 집중 학습용 프로젝트나 개인 포트폴리오 제작 시, 실제 제품 수준의 SNS 기능을 구현할 수 있는 경량 구조의 클론 프로젝트는 매우 효율적이다.

타겟 고객:

프론트엔드 및 풀스택 개발 학습자 (특히 Next.js, Supabase, Clerk 학습 목적)

가벼운 감정 기반 SNS를 경험하고자 하는 일반 사용자

전략적 적합성:

개발자 개인의 학습 효율성을 극대화하고, 향후 개인 포트폴리오 또는 커뮤니티 SNS 확장에 활용할 수 있다.

“감정 기반 마이크로 소셜 플랫폼”이라는 방향성은 MZ 세대의 감정 중심 커뮤니케이션 트렌드와 맞닿아 있다.

핵심 전제/가정:

사용자는 복잡한 SNS보다 단순하고 직관적인 인터페이스를 선호한다.

감정과 순간을 중심으로 한 콘텐츠는 높은 참여율을 유도한다.

제한된 개발 기간(2~3일) 내 MVP 완성이 가능하다.

Proposed Work

솔루션 개요: Instagram의 핵심 UI/UX를 기반으로 하되, 감정 기록 중심의 경량 SNS를 Next.js + Supabase + Clerk 스택으로 구현한다.

핵심 기능:

게시물 작성 (이미지 + 텍스트)

홈피드 (무한 스크롤 기반)

좋아요 / 댓글 상호작용

사용자 프로필 및 통계

Clerk 기반 로그인 및 인증

사용자 플로우:

로그인 또는 회원가입 후 홈피드 진입

다른 사용자의 게시물을 탐색

좋아요 또는 댓글을 통한 상호작용

본인의 감정(사진 + 텍스트) 업로드

프로필에서 자신의 게시물 및 팔로우 상태 확인

**1. 헤더 (60px)**

- 시간: 작고 회색
- ⋯ 메뉴: 우측

- 비율: 1:1 정사각형
- 더블탭 시 좋아요 (모바일)

- 좋아요 수: Bold
- 캡션: 사용자명 Bold + 내용
- 2줄 초과 시 "... 더 보기"
- 댓글 미리보기: 최신 2개만

## 7. 핵심 기능

### 7.1 인증

- **회원가입/로그인**: Clerk 사용
- **정보**: 이메일, 비밀번호, 사용자명
- **리다이렉트**: 로그인 후 홈 (/)

### 7.2 게시물

- **작성**: Sidebar "만들기" → 모달
  - 이미지 업로드 (최대 5MB)
  - 캡션 입력 (최대 2,200자)
- **피드**: 홈 (/)
  - 시간 역순 정렬
  - 무한 스크롤 (10개씩)
- **상세**: 클릭 시 모달 (Desktop) / 페이지 (Mobile)
- **삭제**: ⋯ 메뉴 → 삭제 (본인만)

### 7.3 좋아요

- **위치**: PostCard 하단 ❤️ 버튼
- **상태**: 빈 하트 ↔ 빨간 하트
- **애니메이션**: 클릭 시 scale(1.3) → scale(1)
- **더블탭**: 이미지 더블탭 시 큰 하트 등장

### 7.4 댓글

- **작성**: "댓글 달기..." 입력창, Enter 또는 "게시"
- **표시**:
  - PostCard: 최신 2개 미리보기
  - 상세: 전체 댓글 + 스크롤
- **삭제**: ⋯ 메뉴 (본인만)

### 7.5 프로필

- **내 프로필**: /profile
  - "프로필 편집" (1차 제외, Clerk 설정 사용)
- **다른 사람**: /profile/[userId]
  - "팔로우" 또는 "팔로잉" 버튼
- **게시물 그리드**: 3열, 클릭 시 상세

### 7.6 팔로우

- **버튼 상태**:
  - 미팔로우: "팔로우" (파란색)
  - 팔로우 중: "팔로잉" (회색)
- **Hover**: "언팔로우" (빨간 테두리)
- **클릭**: 즉시 API 호출 → UI 업데이트

## 11. 개발 순서

### 1. 홈 피드 페이지

#### 1-1. 기본 세팅

- Next.js + TypeScript 프로젝트 생성
- Tailwind CSS 설정 (인스타 컬러 스키마)
- Clerk 인증 연동 (한국어 설정)
- Supabase 프로젝트 생성 및 연동
- 기본 데이터베이스 테이블 (users, posts)

#### 1-2. 레이아웃 구조

- Sidebar 컴포넌트 (Desktop/Tablet 반응형)
- MobileHeader 컴포넌트
- BottomNav 컴포넌트
- (main) Route Group 및 레이아웃 통합

#### 1-3. 홈 피드 - 게시물 목록

- PostCard 컴포넌트 (Header, Image, Actions, Content)
- PostCardSkeleton 로딩 UI
- PostFeed 컴포넌트
- /api/posts GET API (페이지네이션)

#### 1-4. 홈 피드 - 좋아요 기능

- likes 테이블 생성
- /api/likes POST/DELETE API
- 좋아요 버튼 및 애니메이션 (하트 + 더블탭)

---

### 2. 게시물 작성 & 댓글 기능

#### 2-1. 게시물 작성 모달

- CreatePostModal 컴포넌트 (Dialog)
- 이미지 미리보기 UI
- 텍스트 입력 필드

#### 2-2. 게시물 작성 - 이미지 업로드

- Supabase Storage 버킷 생성
- /api/posts POST API
- 파일 업로드 로직 및 검증

#### 2-3. 댓글 기능 - UI & 작성

- comments 테이블 생성
- CommentList, CommentForm 컴포넌트
- /api/comments POST API

#### 2-4. 댓글 기능 - 삭제 & 무한스크롤

- /api/comments DELETE API
- 댓글 삭제 버튼 (본인만)
- PostFeed 무한 스크롤 (Intersection Observer)

---

### 3. 프로필 페이지 & 팔로우 기능

#### 3-1. 프로필 페이지 - 기본 정보
